# Практическая работа 2: Модульное приложение с внедрением зависимостей на C#

## Обзор

Данный проект демонстрирует создание модульного приложения на C# с использованием принципов внедрения зависимостей и динамической загрузки модулей. Приложение спроектировано таким образом, что его функциональность может быть расширена без модификации основного ядра, путем добавления новых модулей.

Проект `csharp` содержит следующую структуру:

*   **`csharp/src/Pr2.ModulesAndDi/`**: Основное ядро приложения и демонстрационные модули.
*   **`csharp/tests/Pr2.ModulesAndDi.Tests/`**: Юнит-тесты для ключевой логики приложения.

## Архитектура

Архитектура приложения состоит из двух основных частей: **ядра** и **модулей**.

### Ядро (Core)

Ядро (каталог `csharp/src/Pr2.ModulesAndDi/Core/`) отвечает за:

*   **Определение контракта модуля (`IAppModule`)**: Интерфейс, которому должны соответствовать все модули.
    *   `Name`: Уникальное имя модуля.
    *   `Requires`: Список имен модулей, от которых зависит данный модуль.
    *   `RegisterServices(IServiceCollection services)`: Метод для регистрации служб модуля в контейнере внедрения зависимостей.
    *   `InitializeAsync(IServiceProvider serviceProvider, CancellationToken cancellationToken)`: Асинхронный метод для инициализации модуля и выполнения его основных действий.
*   **Определение контракта действия (`IAppAction`)**: Интерфейс для действий, которые могут быть выполнены модулями.
*   **Загрузка и упорядочивание модулей (`ModuleCatalog`)**: Статический класс.
    *   Обнаруживает модули, реализующие `IAppModule`, из сборок.
    *   Строит корректный порядок запуска модулей с учетом их зависимостей (используя алгоритм топологической сортировки Кана).
    *   Обрабатывает ошибки:
        *   `ModuleLoadException`: Выбрасывается, если требуемый модуль не найден или обнаружена циклическая зависимость.
*   **Точка входа (`Program.cs`)**: Инициализирует конфигурацию, загружает и запускает модули в правильном порядке, управляет DI-контейнером и выполняет действия, предоставляемые модулями.

### Модули (Modules)

Модули (каталог `csharp/src/Pr2.ModulesAndDi/Modules/`) представляют собой расширения функциональности приложения. Каждый модуль:

*   Реализует интерфейс `IAppModule`.
*   Регистрирует свои службы в DI-контейнере через метод `RegisterServices`.
*   Может выполнять определенные действия при инициализации через метод `InitializeAsync` или регистрировать `IAppAction` для последующего выполнения ядром.
*   **Ключевой принцип**: Ядро приложения ничего не знает о конкретной реализации модулей, оно взаимодействует с ними исключительно через контракт `IAppModule`. Это позволяет добавлять, удалять или заменять модули без изменения логики ядра.


## Демонстрационные модули
В проекте реализованы следующие демонстрационные модули (в `csharp/src/Pr2.ModulesAndDi/Modules/`), которые показывают, как модули могут изменять поведение приложения:

*   **`CoreModule`**: Базовый модуль, который предоставляет основные сервисы, такие как `IClock` и `IStorage`. Не имеет зависимостей.
*   **`LoggingModule`**: Добавляет функциональность логирования в консоль. Зависит от `CoreModule`.
*   **`ValidationModule`**: Реализует простую логику валидации данных и сохраняет их в `IStorage`. Зависит от `CoreModule`.
*   **`ExportModule`**: Экспортирует данные из `IStorage` в файл `export.txt`. Зависит от `CoreModule` и `ValidationModule`.
*   **`ReportModule`**: Формирует отчет, используя `IClock` и `IStorage`, и выводит его в консоль. Зависит от `CoreModule` и `ExportModule`.


## Запуск приложения

Чтобы запустить C# приложение:

1.  Перейдите в директорию `csharp/src/Pr2.ModulesAndDi/`.
2.  Откройте терминал и выполните команду:
    ```bash
    dotnet run
    ```

## Запуск тестов

Юнит-тесты находятся в проекте `csharp/tests/Pr2.ModulesAndDi.Tests/`. Чтобы запустить тесты:

1.  Перейдите в директорию `csharp/tests/Pr2.ModulesAndDi.Tests/`.
2.  Откройте терминал и выполните команду:
    ```bash
    dotnet test
    ```
